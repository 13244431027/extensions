扩展介绍

本扩展为 Scratch 项目提供一个完整的屏幕虚拟键盘，可在触摸屏设备、平板电脑或任何需要屏幕输入的场景下替代物理键盘。键盘以浮动窗口形式覆盖在项目上方，支持拖动、缩放，并完整模拟真实键盘事件。

核心特性

🔤 字母键盘
标准 QWERTY 布局，含 Esc、修饰键、空格等
🔢 数字+方向+功能
F1-F12、方向键、小键盘、导航键

🔒 锁定键

积木说明

本扩展共提供 9 个积木，分为显示控制、模式切换、状态查询和模拟控制四类。

显示控制

显示虚拟键盘

显示虚拟键盘

直接显示虚拟键盘。如果键盘已隐藏，则以默认的「字母键盘」模式打开并居中于屏幕；如果已显示则不做改变。

隐藏虚拟键盘

隐藏虚拟键盘

关闭虚拟键盘窗口，同时释放所有当前按下的按键（包括粘性修饰键和锁定键状态会被清除）。

条件显示/隐藏

[显示] 虚拟键盘

通过下拉菜单选择「显示」或「隐藏」来控制键盘。菜单选项固定，不接受变量输入。

模式切换

显示指定类型键盘

显示虚拟键盘 类型 [字母键盘]

显示虚拟键盘并同时切换到指定类型。菜单提供三种选项：字母键盘、数字+方向+功能、游戏键盘(仅方向)。若键盘已显示，则仅切换类型。

切换键盘类型

切换虚拟键盘为 [字母键盘]

在键盘已显示的情况下切换布局类型。如果键盘当前未显示，此积木不会自动显示键盘（仅设置内部模式）。

状态查询

键盘可见性

虚拟键盘已显示？

返回布尔值（true 或 false），表示虚拟键盘窗口当前是否在屏幕上可见。

当前键盘类型

当前虚拟键盘类型

返回一个字符串，表示当前键盘模式。可能的返回值：字母键盘、数字+方向+功能、游戏键盘。

模拟控制（新增）

模拟按键开关

模拟按键 [启用]

控制虚拟键盘点击时是否向 Scratch 项目发送真实的键盘事件。选择「启用」时，每次点击都会触发当按下某键等积木；选择「停止」时，按键视觉上仍会高亮，但不会产生实际输入效果。

默认状态为启用。此开关在调试、教学演示或需要暂时冻结输入的场合非常实用。关闭模拟后，已按下的按键状态仍可通过「按下了那些按键」积木查询。

查询按下按键

按下了那些按键

返回一个字符串，包含当前虚拟键盘上所有处于按住状态的按键名称，多个按键以逗号分隔。如果没有按键被按住，则返回无。

返回值示例：a, shift, 3 或 ArrowUp, ArrowRight。此积木可配合「模拟按键 停止」使用，在不触发事件的情况下检测用户操作。

补充说明

修饰键与锁定键行为

  • 粘性修饰键：Shift、Ctrl、Alt、Meta（含左右版本）在被点击后会保持激活状态（高亮），再次点击则释放。适合需要组合键的场景。
  • 锁定键：CapsLock、NumLock、ScrollLock 为切换式按键，点击一次锁定、再点击一次解锁，状态会持续保留直到再次切换。
  • 窗口失焦：当浏览器窗口失去焦点时，所有按键自动释放，防止按键卡住。
  • 长按连发：普通字符键和功能键支持长按连发（延迟 420ms，间隔 75ms）；修饰键和锁定键不触发连发。

窗口操作

  • 拖拽标题栏可移动键盘到屏幕任意位置。
  • 键盘自动根据窗口尺寸缩放，最小缩放比例为 0.35。
  • 点击右上角 × 按钮可关闭键盘，等同于调用「隐藏虚拟键盘」积木。
  • 点击标题栏中的模式按钮（字母键盘 / 数字/方向 / 游戏键盘）可快速切换布局。

兼容性提示

  • 本扩展仅在非沙盒模式下可用。若在沙盒环境中加载，会抛出错误。
  • 扩展通过 KeyboardEvent 和 Scratch VM 的 ioDevices.keyboard.postData 双通道发送按键事件，确保 Scratch 积木能正确响应。
  • 在移动端触摸屏上，建议配合游戏键盘模式使用以获得最佳体验。
